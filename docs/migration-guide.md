# Migration Guide

Use this guide to move improvements from one project back into this template.

## Good Candidates For The Template

- General AI working rules.
- Human approval gates.
- Spec templates.
- Validation workflows.
- Reusable architecture lessons.
- Decision record patterns.
- Cross-project skill conventions.

## Keep Project-Specific

- Product names.
- Game-specific mechanics.
- Private service endpoints.
- Team-only credentials or URLs.
- Highly specific file paths.
- Generated assets.

## Porting Checklist

1. Remove project names and replace with placeholders.
2. Move domain-specific rules into `examples/` if not universal.
3. Keep templates short and reusable.
4. Update `README.md` if installation or usage changes.
5. Tag a new version when the template is stable.
